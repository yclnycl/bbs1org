# migrate 数据库迁移目录

本目录存放论坛数据库的增量迁移脚本，用于在不重装的情况下逐步升级已有数据库结构或初始数据。

## 文件命名规范

迁移文件统一命名为：

```
migrate_YYYY_MM_DD_NN.php
```

- `YYYY_MM_DD`：日期（年_月_日）。
- `NN`：当日序号，两位数字，从 `01` 开始（如 `migrate_2026_06_30_01.php`、`migrate_2026_06_30_02.php`）。

## 执行与锁文件

迁移脚本由 `install.php?do=migrations` 统一调度执行。执行器会建立数据库连接得到 `$db`，然后 `include` 每个待执行的迁移文件；迁移文件内部直接使用 `$db` 变量即可，不要自己连接数据库。

每个迁移文件成功执行后，会在 `data/` 目录下生成同名的 `.lock` 锁文件，路径为：

```
data/<迁移文件名>.lock
```

例如执行 `migrate_2026_06_30_01.php` 后会生成 `data/migrate_2026_06_30_01.php.lock`。

判断一个迁移是否「待执行」的规则是：**迁移文件存在 且 对应的 `.lock` 不存在**。只要 `.lock` 存在，该迁移就不会被重复执行。

## 如何新增迁移

1. 复制一个已有的迁移文件作为模板（例如本目录中的 `migrate_2026_06_30_0X.php`）。
2. 按命名规范改名，日期使用当天，序号取当日下一个未用编号。
3. **必须保持幂等**：同一迁移重复执行不应报错、不应破坏数据。
   - SQLite 不支持 `ADD COLUMN IF NOT EXISTS`，新增列前请用 `PRAGMA table_info(表名)` 检查列是否已存在。
   - 建表用 `CREATE TABLE IF NOT EXISTS`。
   - 插入设置用 `INSERT OR IGNORE` 等方式避免重复插入。
4. 文件内只声明对 `$db` 的操作，不要输出 HTML、不要 `echo`、不要 `?>` 闭合标记、不要自己连接数据库。

## 参考

- 执行入口：`install.php?do=migrations`
- 数据库连接、锁文件写入由执行器负责。
